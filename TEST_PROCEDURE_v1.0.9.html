Test Procedure — RS Polymorph v1.0.9

Date: 2026-05-05
Target environment: Minecraft 1.21.1 · NeoForge 21.1.219+ · Refined Storage 2 ≥ 2.0.1 · Polymorph ≥ 1.1.0 · Create 6.0.10 (for the regression test below)
Test scope: singleplayer (integrated server) + dedicated server (multiplayer).
Issue: #1 — Incompatibility/Crash with Create (root-cause fix on top of the 1.0.8 backstop).

1. Encased fan crash — root-cause fix verification

Reproduces the exact setup from issue #1: a powered encased fan blowing through lava, smelting items dropped in front of it. Before 1.0.9, our factory leaked RsGridRecipeData onto every block entity in the world, polluting Polymorph's input-keyed cache across recipe types.

2. Factory class-isolation check

Verifies that RsGridRecipeData is no longer attached to non-RS2 block entities.

3. Pattern Grid — recipe selection (1.0.7+ regression check)

4. Crafting Grid — recipe selection (1.0.7+ regression check)

5. Autocrafting — tagged pattern resolution (regression)

6. Server-only integrity (regression)

7. Save migration (worlds created on 1.0.7 / 1.0.8)

Worlds created with 1.0.7 or 1.0.8 may have stale polymorph:recipe_data attachments on non-RS2 BEs (saved into chunk NBT). 1.0.9 stops creating new ones, but existing ones persist until the chunk is unloaded.

8. Known issues / regressions to watch

Procédure de test — RS Polymorph v1.0.9 (Français)

Date : 2026-05-05
Environnement cible : Minecraft 1.21.1 · NeoForge 21.1.219+ · Refined Storage 2 ≥ 2.0.1 · Polymorph ≥ 1.1.0 · Create 6.0.10 (pour le test de régression ci-dessous)
Portée des tests : solo (serveur intégré) + serveur dédié (multijoueur).
Issue : #1 — Incompatibility/Crash with Create (correctif racine au-dessus du filet de sécurité 1.0.8).

1. Crash du ventilateur encastré — vérification du correctif racine

Reproduit le setup exact de l'issue #1 : un ventilateur encastré motorisé qui souffle à travers un bloc de lave, smeltant les items déposés devant. Avant 1.0.9, notre factory laissait fuiter RsGridRecipeData sur chaque block entity du monde, polluant le cache de Polymorph (indexé par input uniquement) entre les types de recettes.

2. Vérification de l'isolation des factories par classe

Vérifie que RsGridRecipeData n'est plus attaché aux block entities non-RS2.

3. Pattern Grid — sélection de recette (régression 1.0.7+)

4. Crafting Grid — sélection de recette (régression 1.0.7+)

5. Autocraft — résolution du patron tagué (régression)

6. Intégrité serveur (régression)

7. Migration de sauvegardes (mondes créés en 1.0.7 / 1.0.8)

Les mondes créés avec 1.0.7 ou 1.0.8 peuvent avoir des attachments polymorph:recipe_data obsolètes sur des BEs non-RS2 (sauvegardés dans le NBT du chunk). 1.0.9 arrête d'en créer mais ceux qui existent persistent jusqu'au déchargement du chunk.

8. Points de vigilance / régressions à surveiller
